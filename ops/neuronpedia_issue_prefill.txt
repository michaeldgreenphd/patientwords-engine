jlens: gemma-2-2b and gemma-2-2b-it return identical readouts - one host behind both ids?

Thanks for shipping the hosted Jacobian lens - we adopted it within the week for a
clinical-vs-colloquial language study (https://michaeldgreenphd.github.io/patientwords/answer-depth/).

**Observation.** `POST /api/lens/prompt` with `type: ["JACOBIAN_LENS"]`, `topN: 8`,
`numCompletionTokens: 0`, `stream: false` returns identical `top_tokens` for
`modelId: "gemma-2-2b"` and `modelId: "gemma-2-2b-it"` - byte-identical rankings at
every layer and position across 100 prompts (50 sentence pairs x 2 phrasings),
measured 2026-07-11. `meta.model` echoes the requested id in both cases.

Base and instruct models should not agree to that precision, so we suspect both ids
route to one lens/inference host. Committed artifacts (summaries + raw responses):
https://github.com/michaeldgreenphd/patientwords-engine/tree/main/trace_out
(`pairs_20260711T051145Z__jlens_gemma-2-2b` vs `__jlens_gemma-2-2b-it`).

**Question.** Are separate lens hosts deployed for the two ids? If not, is the
instruct lens planned? We currently report one model on our site pending this.

Happy to provide exact request bodies or re-run against a staging host.
